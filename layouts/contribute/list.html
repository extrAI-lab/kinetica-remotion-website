Contribute a component

A component is a self-contained animated scene — users install one or more and they are sequenced automatically into a single video. Follow the steps below to get yours merged.

1

Copy the component template

Copy src/components/_template/ and rename it to your component slug (kebab-case).

src/components/my-component/
├── MyComponent.tsx # main component
├── schema.ts # Zod schema + defaults
├── index.ts # re-exports
└── preview.png # first-frame render
2

Define the prop schema

Every visual property (color, text, size, speed) must be a Zod-validated prop with a default. No hardcoded values inside the component. The *Defaults export must include every prop — it is used as-is by the CLI when scaffolding.

// schema.ts export const myComponentSchema = z.object({ backgroundColor: z.string().default('#020617'), text: z.string(), }); export const myComponentDefaults: MyComponentProps = { backgroundColor: '#020617', color: '#f8fafc', text: 'Hello', };
3

Generate the preview

Run the render command to produce preview.png and commit it with your PR.

pnpm render:preview my-component
4

Add the manifest entry

Add your component to src/components/manifest.json under the components key. The defaultProps must exactly match your *Defaults export.

{ "my-component": { "title": "My Component", "description": "One sentence describing what it looks like.", "category": "3d", "durationInFrames": 120, "fps": 30, "width": 1920, "height": 1080, "componentName": "MyComponent", "schemaName": "myComponentSchema", "defaultsName": "myComponentDefaults", "defaultProps": { /* copy from your *Defaults export */ }, "files": [ "src/components/my-component/MyComponent.tsx", "src/components/my-component/schema.ts", "src/components/my-component/index.ts" ] } }
5

Register for Studio

Update src/kinetica-components.json (append your slug to the components array) and mirror the same change in src/Root.tsx — import the component, add a <Series.Sequence> inside KineticaMainVideo, and a <Composition> under the components folder. Update the top-level durationInFrames to the sum of all components.

Acceptance checklist

Prop schema

schema.ts defines a Zod schema with every visual property exposed as a prop. No hardcoded colors, strings, or sizes.

Demo data

schema.ts exports a *Defaults object with realistic values that produce a good-looking preview without any edits — all props must be present, no undefined optional fields.

Self-contained

A component must not import from other components. Any shared animation logic must be inlined as a private local function within the component file.

Preview render

A preview.png (first frame) lives in the component folder and is committed with the PR.

Manifest entry

src/components/manifest.json has an entry with the correct files, defaultProps, category, and metadata.
