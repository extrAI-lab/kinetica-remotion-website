Contribute a component

A component is a self-contained Remotion composition — users install one or more and they are sequenced automatically into a single video. Follow the steps below to get yours merged.

1

Copy the component template

Copy src/scenes/_template/ and rename it to your component slug (kebab-case).

src/scenes/my-scene/
├── MyScene.tsx # main component
├── schema.ts # Zod schema + defaults
├── index.ts # re-exports
└── preview.png # first-frame render
2

Define the prop schema

Every visual property (color, text, size, speed) must be a Zod-validated prop with a default. No hardcoded values inside the component.

// schema.ts export const mySceneSchema = z.object({ backgroundColor: z.string().default('#020617'), text: z.string(), }); export const mySceneDefaults: MySceneProps = { backgroundColor: '#020617', color: '#f8fafc', text: 'Hello', };
3

Generate the preview

Run the render command to produce preview.png and commit it with your PR.

pnpm render:preview my-scene
4

Add the manifest entry

Add your component to src/scenes/manifest.json. The defaultProps must exactly match your *Defaults export.

{ "my-scene": { "title": "My Scene", "description": "One sentence describing what it looks like.", "durationInFrames": 120, "fps": 30, "width": 1920, "height": 1080, "componentName": "MyScene", "schemaName": "mySceneSchema", "defaultsName": "mySceneDefaults", "defaultProps": { /* copy from your *Defaults export */ }, "files": [ "src/scenes/my-scene/MyScene.tsx", "src/scenes/my-scene/schema.ts", "src/scenes/my-scene/index.ts" ] } }

Acceptance checklist

Prop schema

schema.ts defines a Zod schema with every visual property exposed as a prop. No hardcoded colors, strings, or sizes.

Demo data

schema.ts exports a *Defaults object with realistic values that produce a good-looking preview without any edits — all props must be present, no undefined optional fields.

Self-contained

A component must not import from other components. Any shared animation logic must be inlined as a private local function within the component file.

Preview render

A preview.png (first frame) lives in the component folder and is committed with the PR.

Manifest entry

src/scenes/manifest.json has an entry with the correct files, defaultProps, and metadata.
