Why kinetica

Stop building animations from scratch.

Every component is production-ready: typed props, sensible defaults, and a single CLI command to drop it into any Remotion project.

⚡

Ship in minutes

One command scaffolds a full Remotion project with your chosen components already wired up, ready to render.

🎛

Fully customizable

Every visual property — colors, text, timing, sizes — is exposed as a typed Zod prop. Tweak in Remotion Studio or pass data programmatically.

📦

Self-contained

Each component has zero cross-dependencies. Copy one or ten — they never interfere with each other or your existing codebase.

🔁

Composable by design

Components chain automatically via Remotion's <Series> — pick multiple and they become a single, sequenced video.

🏷

Typed end-to-end

Full TypeScript and Zod schemas mean your editor catches mistakes before Remotion ever renders a frame.

🌱

Open & growing

The library is open-source and accepting contributions. New components ship as standalone packages — no breaking changes.

How it works

From zero to rendered video in three steps.

No boilerplate, no manual wiring. The CLI handles project setup so you can focus on customizing.

1

Pick your components

Browse the library and select one or more components. Each card shows a live preview so you know exactly what you're getting.

2

Run the CLI

Copy the generated npx create-kinetica install command. It scaffolds a Remotion project with your components already sequenced.

3

Customize & render

Open Remotion Studio, edit props visually, then render to MP4. Every prop is live-editable — no code changes required for simple tweaks.
